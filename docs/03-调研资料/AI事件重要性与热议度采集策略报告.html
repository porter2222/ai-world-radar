Research Report

AI 事件重要性与热议度采集策略报告

本报告基于 AI World Radar 项目 docs，重新定义采集目标：系统要发现的是有主体、有触发点、有来源、有中文用户价值的 AI 事件，而不是单纯抓最热门的链接。

更新时间：2026-06-05 P1 / P1.5 / P2 策略 事件价值优先

1. 报告目的

用户关心的不只是“最火爆”，而是 AI World Radar 需要抓哪些信息，以及如何同时识别“最热议”和“最重要”的 AI 事件。

采集系统的目标不是抓更多网页，而是把多源信号变成可回源、可聚合、可排序、可解释的 AI 事件。

2. 从项目 docs 抽出的信息需求

AI World Radar 的核心内容单位是事件，不是新闻、帖子、视频或仓库。候选事件必须能说成一句：谁或什么东西，发生了什么变化。

目标信息含义示例
主体事件围绕谁或什么OpenAI、Claude、GitHub Copilot、某个开源仓库
触发点发生了什么变化发布、更新、涨价、开源、争议、政策变化
来源能否回源官方博客、GitHub、HN、HF、YouTube、Reddit
热度线索是否正在被讨论排名、评论数、增速、多平台重复出现
重要度线索为什么即使不热也重要重大模型、核心论文、平台能力、安全政策
中文用户价值为什么目标用户需要知道学习认知、工具使用、开发实践、风险理解

3. 不应该只追“火爆”

只追热度会导致娱乐化、漏掉低热高影响事件、重复内容污染首页。项目当前排序口径应是：热度主导 + 重要度兜底 + 中文用户价值修正。

线索类型目标典型来源最终去向
热议线找正在发酵的讨论HN、GitHub Trending、HF Trending、Reddit、YouTube、X热议型候选事件
重要线找影响大但未必爆火的事件官方博客、GitHub Changelog、论文、安全公告事实型候选事件

4. 四档分流模型

分流进入条件处理方式
事实型候选事件官方、一手页面、GitHub、HF、论文页出现明确主体和触发点进入事件生成链路
热议型候选事件外网讨论热度高，能反映趋势、争议或舆论变化谨慎表达为社区热议或待确认
待核验线索主体明确但来源不足、变化不清或热度不够不上首页，继续补源观察
丢弃内容教程、工具合集、营销软文、旧闻、低相关内容不进入 Agent 生成链路

5. 来源分层

P1 不追求全网覆盖，应优先用稳定、可回源、低风险的来源跑通事件生产闭环。

来源层作用P1 是否适合示例
事实源支撑正式事件适合OpenAI News、Anthropic News、DeepMind Blog、NVIDIA Blog、GitHub Changelog
热度源判断外网是否关注P1 简单源，P1.5 扩展HN、GitHub Trending、HF Trending、Reddit、YouTube、X
线索源补盲和冷启动可用但不能当唯一事实依据TLDR AI、AIBase、newsletter、竞品聚合站

6. 热度算法

热度不应只看绝对数字。更适合使用平台位置、互动强度、增长速度、爆发检测、跨源共振和讨论质量。

heat_score =
  0.30 * platform_rank_score
  + 0.25 * engagement_score
  + 0.20 * velocity_score
  + 0.15 * cross_source_score
  + 0.10 * discussion_quality_score
  • 平台位置：HN topstories、GitHub Trending、HF Trending、Reddit hot/rising。
  • 互动强度：score、comments、likes、views、stars today、downloads。
  • 增长速度：当前指标减上一轮指标，再除以时间差。
  • 爆发检测：当前增速相对来源历史基线的异常程度。
  • 跨源共振：同一事件出现在多个独立来源。

7. 重要度算法

重要度用于兜底“低热但必须知道”的事件。它由主体重要度、事件类型、来源质量、影响范围和新颖性组成。

impact_score =
  0.30 * subject_tier_score
  + 0.25 * event_type_score
  + 0.20 * source_quality_score
  + 0.15 * scope_score
  + 0.10 * novelty_score

P1 最新技术口径不做真实性评分。这里的 source_quality 只表示来源作为线索或证据的优先级，不代表事实核验完成。

8. 综合排序模型

ranking_score =
  0.40 * heat_score
  + 0.30 * impact_score
  + 0.20 * audience_value_score
  + 0.10 * freshness_score
  + evidence_bonus
  - risk_penalty

同时需要两个硬规则：高重要度 + 高来源质量的事件可以低热兜底；高热但来源差的内容只能作为热议或待确认谨慎处理。

10. 推荐落地顺序

  1. P1 第一切片：HN AI 事件生产闭环。
  2. P1 第二切片：OpenAI、Anthropic、DeepMind、NVIDIA、GitHub Changelog 官方事实源。
  3. P1 第三切片：GitHub watchlist / Hugging Face Models / Papers / Trending。
  4. P1.5：GitHub Trending、Reddit 指定社区、YouTube 指定频道、HN Algolia query profile、changedetection。
  5. P2：X 官方 API、YouTube 字幕/评论、Facebook Graph API 授权页面、Browser fallback。

raw_heat_metrics:
  platform_rank: 12
  score: 180
  comments: 75
  stars_today: null

heat_clues:
  - HN topstories rank 12
  - 75 comments in engineering discussion

impact_clues:
  - affects developer workflow
  - official source attached

suggested_route: high_heat_candidate
candidate_score: 0.78

12. 质量门禁

  • 有可点击来源。
  • 标题和正文一致。
  • 未确认内容不写成事实。
  • 社区热议事件标明讨论性质。
  • 详情页不是字段堆叠。
  • 不直接搬运竞品正文。
  • 不生成不存在的链接、截图、数据或人物言论。
  • 内容不足时少发，不强行凑 Top 12。

13. 最终建议

Source Registry
-> Adapter Fetch
-> SourceItem
-> EvidenceCard
-> Four-way Route
-> EventCluster
-> Heat / Impact / Audience Ranking
-> Quality Gate
-> PublishedEvent / Brief
热度负责发现正在发生的讨论，重要度负责防漏关键事件，中文用户价值负责决定它是否值得展示给目标用户。
