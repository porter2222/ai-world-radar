Backend Agent Brief

后端 P1 开发代理任务书

本文档是给后端开发代理使用的可执行任务书，定义后端代理在第一轮工程骨架开发中的职责、边界、阶段、测试证据和提交规则。

版本：v0.1 更新时间：2026-06-09 适用对象：后端开发代理

1. 任务书目的

后端开发代理拿到本文档后，应能在不重新讨论产品方向的前提下，完成后端 P1 第一轮工程骨架开发，并清楚知道需要先读哪些文档、允许修改哪些目录、禁止做哪些事情、每个阶段交付什么、每个阶段如何验证、测试证据如何展示给用户，以及最终如何提交和交接。

2. 后端代理角色定位

负责 AI World Radar 后端 P1 工程底座和 HN 事件生产闭环的工程实现者。

后端代理负责把管道搭出来，但不负责设计最终 Agent 智能逻辑。

后端代理负责 apps/worker 工程结构、Python 依赖、PostgreSQL / SQLAlchemy / Alembic、HN Algolia collector、去重、热度预排序、original_url 原文抓取与本地缓存、PipelineRun、Agent stub、pipeline 串联、txt 运行报告、本地运行入口和第一轮测试。

后端代理不负责真实 Agent prompt 调优、最终智能逻辑、前端页面、FastAPI、部署、产品定位修改和真实性核验系统。

5. 职责边界

后端代理只做工程底座和管道，不做真实 Agent 智能逻辑。后端代理应提供 Agent 节点接口和 stub，使 pipeline 能跑通。后续 Agent 代理会把 stub 替换为真实 prompt、schema 和 LLM 调用。

9. Agent Stub 口径

EvidenceAgentStub
-> 根据 HN title / page_title 返回简单 EvidenceCard JSON

EventClusterAgentStub
-> 默认 1 个 EvidenceCard 生成 1 个 EventCluster

RankingAgentStub
-> 使用 hn_heat_score 生成简单 ranking_score

DetailWriterAgentStub
-> 用 title + excerpt 拼一个临时详情

BriefWriterAgentStub
-> 从 top PublishedEvent 拼一个临时 brief

QualityGateAgentStub
-> 检查字段齐全就 publish

stub 必须输出结构化数据，方便后续 Agent 代理替换。

11. 真实网络测试策略

真实网络测试不禁止，但必须和稳定单元测试分开。

  • 普通单元测试优先使用 fixture / mock，保证稳定。
  • 真实网络测试必须使用 pytest marker 标记为 network。
  • 真实网络测试可以请求 HN Algolia API。
  • 真实网页抓取测试可以少量存在，但必须控制数量和超时。
  • 真实 LLM 调用不属于后端代理第一轮测试范围。
pytest
pytest -m network

12. 测试证据与可见性要求

测试项测试文件/命令测试数据验证什么结果

热度分测试pytest tests/test_hn_scoring.pypoints/comments 样例分数是否正确通过

最终报告必须包含测试命令、测试数量、通过数量、失败数量、跳过数量、失败原因和未覆盖范围。

14. 阶段化开发步骤

阶段 1：工程脚手架

建立 apps/worker、依赖文件、.env.example、基础配置读取、runtime 目录说明或 .gitkeep。

阶段 2：数据库与迁移

实现 SQLAlchemy session、P1 核心 models、Alembic 初始化和第一版 migration。

阶段 3：HN collector

实现 HN Algolia 请求封装、query profile、7 天时间窗、limit 100、scoring 和 dedupe。

阶段 4：原文抓取与缓存

实现 original_url 可读正文抓取、runtime/source-pages/ 写入、hash、excerpt、cache path、超时和失败记录。

阶段 5：Agent stubs 与 pipeline 串联

实现确定性 Agent stubs、串起 HN pipeline、写入数据库和幂等逻辑。

阶段 6：运行报告与最终验收

生成终端报告和 runtime/pipeline-reports/*.txt，更新测试记录文档并运行测试。

16. 最终验收标准

  1. apps/worker 工程结构存在。
  2. 依赖文件和 .env.example 存在且无真实 key。
  3. Alembic migration 存在。
  4. HN collector、scoring、dedupe、original_url 缓存、Agent stubs、pipeline report 都有测试。
  5. scripts/run_hn_pipeline.py 存在。
  6. runtime/pipeline-reports/ 策略存在。
  7. docs/07-验收与运行/后端P1测试记录.md 存在。
  8. 最终回答列出测试证据。

17. 提交与推送策略

阶段检查点报告 + 最终一次 commit/push。

最终提交前必须跑测试、报告测试结果、检查 git status，确认没有提交 .env、runtime 真实产物或 API Key。

porter <2037388336@qq.com>

如果中途遇到阻塞，不要提交半成品，应输出阻塞原因、已完成内容和当前状态。
