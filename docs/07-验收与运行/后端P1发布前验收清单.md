## 8. 是否可以进入产品接口层

本地后端 Worker 能力可以进入产品接口层设计和开发准备。

实际实现前仍需注意：

1. P1-6 已确认采用 `ProductQueryService + FastAPI` 薄适配层。
2. 远程 push 阻塞仍需在 GitHub 443 网络恢复后重试，确保远程分支包含 `b75fecc` 及后续文档提交。

产品接口层最小范围建议是：

1. `GET /events`：读取已发布事件列表。
2. `GET /events/{slug}`：读取事件详情。
3. `GET /pipeline-runs/{id}`：读取 pipeline run 结果。
4. `GET /agent-runs?pipeline_run_id=...`：读取 agent 审计记录。
5. `GET /review-queue`：读取 `manual_review` 队列。

本阶段不建议直接做前端 UI，先把产品接口契约、查询服务和最小 API 测试补齐。
