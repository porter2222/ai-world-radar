# 本地开发调试指南

更新时间：2026-06-09

## 1. 文档目的

本文档用于解决在 VSCode 或终端中手动测试某个函数、某个 Agent 节点、某个脚本时常见的导包问题。

核心原则：

> `worker` 是一个 Python 包，应通过虚拟环境的可编辑安装进入解释器，而不是依赖当前脚本所在目录碰运气导包。

## 2. 一次性环境准备

在 PowerShell 中执行：

```powershell
cd "D:\AI World Radar\apps\worker"
.\.venv\Scripts\python -m pip install -e ".[dev]"
```

完成后，无论你从项目根目录还是 `apps/worker` 目录运行脚本，都可以正常导入：

```python
from worker import LLMClient
from worker.agents.detail_writer_agent import DetailWriterAgentStub
```

## 3. VSCode 设置

项目已经提供：

```text
.vscode/settings.json
.vscode/launch.json
```

```text
D:\AI World Radar

选择 D:\AI World Radar\apps\worker\.venv\Scripts\python.exe
```

然后执行：

```text
Developer: Reload Window
```

## 4. 临时测试脚本放哪里

临时脚本统一放：

```text
apps/worker/scratch/
```

例如：

```text
apps/worker/scratch/test_detail_writer.py
```

运行方式：

```powershell
cd "D:\AI World Radar"
.\apps\worker\.venv\Scripts\python apps\worker\scratch\test_detail_writer.py
```

这些临时脚本默认不会提交到 Git。

不要把临时脚本直接放在 `apps/worker` 根目录。根目录下的临时文件会出现在 Git 未跟踪列表里，后续提交时容易误选。

## 5. 推荐的临时脚本模板

```python
from worker import LLMClient


def main():
    client = LLMClient()
    print(client.provider)
    print(client.model)


if __name__ == "__main__":
    main()
```

## 6. 什么时候写 pytest

临时脚本只用于探索。

如果你确认某个行为以后还需要反复验证，应把它转成：

```text
apps/worker/tests/test_xxx.py
```

并运行：

```powershell
cd "D:\AI World Radar\apps\worker"
.\.venv\Scripts\python -m pytest
```

## 7. 真实 LLM 调用

真实 API Key 只放在项目根目录 `.env` 中，不要写入临时脚本、测试文件、文档或提交记录。

测试 LLMClient 配置：

```powershell
cd "D:\AI World Radar\apps\worker"
.\.venv\Scripts\python scripts\smoke_llm_client.py
```

真实调用模型：

```powershell
.\.venv\Scripts\python scripts\smoke_llm_client.py --call
```

注意：真实调用会消耗 API 额度。
