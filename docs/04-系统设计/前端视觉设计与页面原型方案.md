# 前端视觉设计与页面原型方案

更新时间：2026-06-23

## 1. 当前目标

本阶段先把 AI World Radar 的公共侧 Web 视觉方向和核心阅读体验定住，不做前后端联调，不接真实 API，不开发后台页面。

当前高保真 HTML 原型只覆盖 Web 桌面端：

- 首页事件流。
- 事件详情阅读区。
- 桌面端双栏交互：左侧事件流，右侧事件档案。

暂时不做移动端，也不定义移动端事件流到详情页的切换交互。

原型文件：

- `docs/04-系统设计/前端视觉设计与页面原型方案.html`

## 2. 设计判断

AI World Radar 当前不应该做营销落地页，也不应该做复杂筛选型资讯站。首页只保留用户真正需要的事件流，让用户快速判断“这件事是什么、值不值得点开”。

详情页不做普通新闻摘要，也不做报告式堆字段。它应该像轻量事件档案：自然正文为主，补充“为什么值得看”“继续观察”“来源”三类增量信息。

公共页面不展示：

- SourceSignal / EventDossier / PublishedEvent 等内部模型名。
- Agent、pipeline、review、已复核等开发或后台状态。
- 热度分数、排序解释、调试字段。
- 今日简报、筛选栏、分类入口。

## 3. 视觉系统

本次视觉系统已同步到项目根目录：

- `PRODUCT.md`
- `DESIGN.md`

核心原则：

- 白底、清晰文字、克制色彩。
- 统一 8px 卡片圆角。
- 轻边框代替大阴影。
- 事件卡不塞满解释。
- 详情正文保持自然中文阅读。
- 图片位必须存在，后续接入 `cover_image_url` 或内容生成图兜底。

## 4. 原型交互

Web 桌面端：

- 左侧为事件流。
- 右侧为当前选中的事件详情。
- 点击事件卡后，右侧事件档案即时切换。
- 右侧始终有内容，不保留空白占位栏。

## 5. 后续联调边界

等后端接口和前端工程入口稳定后，第一批只接：

- `GET /events`
- `GET /events/{slug}`

当前 HTML 原型中的字段命名已经贴近后端产品接口，但数据仍是静态样张。
