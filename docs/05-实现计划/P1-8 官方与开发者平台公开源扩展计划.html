P1-8 平台公开源扩展

目标 边界 候选探测 任务拆解 验收标准

目标

在 P1-7 official_feeds 基础上扩展更多低成本公开平台源，让更多官方和开发者平台 feed 稳定进入 source_signals。

RSS firstNo anti-botSourceSignal only

本阶段继续沿用 SourceSignal -> EventCandidate -> EventDossier -> ReviewResult -> PublishedEvent 主链路。新增 profile 不直接运行 Agent、不直接发布、不新增数据库表。

启动基线：108 passed in 31.45s。

阶段边界

负责

  • 新增或修正公开 RSS/XML profile。
  • 补 profile 稳定性测试和 fixture smoke。
  • 更新测试记录、项目状态、文档索引和 README。

不负责

  • 不接 X、Reddit、YouTube、Facebook、TikTok。
  • 不做账号池、代理池、验证码绕过或浏览器反爬。
  • 不做论文源和深层爬取。

验收标准

  • 新增 profile 均有测试覆盖。
  • fixture mode 多 profile 采集仍只写 sources / source_signals。
  • 不创建 pipeline_runs 或 published_events。
  • 测试记录包含真实命令、真实输出、失败修复和未覆盖范围。
