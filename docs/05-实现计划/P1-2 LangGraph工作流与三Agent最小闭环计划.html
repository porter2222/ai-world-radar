Goal: 在 P1-1 新版数据底座之上，用 LangGraph 和确定性三 Agent stub 跑通事件档案生产与发布的最小工作流闭环。

1. 当前阶段

P1-1 已完成新版后端数据底座。P1-2 已完成 LangGraph 工作流骨架，不恢复旧 HN pipeline，也不直接接真实 LLM，而是把 P1-1 的数据底座放进可运行的最小事件生产闭环里。

2. 阶段边界

负责

  • 引入 langgraph==1.2.4。
  • 定义 workflow state schema。
  • 新增值班编辑、研究写作、审稿发布三类确定性 Agent stub。
  • 建立工程 tool 适配层，封装 P1-1 服务层。
  • 实现 LangGraph event pipeline 和新版脚本入口。
  • 对旧 run_hn_pipeline.py 做 legacy guard。
  • 更新测试记录、项目状态、文档索引和 HTML 阅读版。

不负责

  • 不接真实 LLM Agent、prompt、repair prompt 或 tool-calling。
  • 不做 HN / GitHub 真实采集接入，这属于 P1-3。
  • 不开发 FastAPI、Next.js 前端、后台 UI、Redis、队列、对象存储、向量数据库。
  • 不恢复旧 EvidenceCard / EventCluster / Brief 主链路。

5. 验收标准

  • langgraph==1.2.4 是 worker 直接依赖。
  • EventPipelineState 能表达 run、signals、candidate、dossier、review、published event、当前节点、修订次数和最终状态。
  • 三 Agent stub 输出 P1-1 schema 兼容 payload。
  • LangGraph workflow 首跑可以从预置 SourceSignal 生成 PublishedEvent。
  • pipeline_runs 计数字段与最终入库结果一致。
  • agent_runs 至少记录 editor、writer、reviewer 三类 stub 调用。
  • 新脚本入口是 scripts/run_event_pipeline.py，旧入口有明确 legacy guard。
  • worker 全量 pytest 通过，测试记录写清真实命令、真实输出、测试数据、失败修复和未覆盖范围。
