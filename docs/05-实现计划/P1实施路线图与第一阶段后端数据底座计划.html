P1 实施路线图与第一阶段后端数据底座计划

版本：v1.0 ｜ 更新日期：2026-06-12 ｜ 阶段：Superpowers implementation plan

For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox syntax for tracking.

Goal: 将 AI World Radar P1 从旧的 HN 事件闭环骨架，重构为新版“事件档案生产与发布”架构的后端数据底座。

Architecture: 本计划先给出 P1 总路线图，再只执行第一阶段。第一阶段只做 Python Worker 的数据模型、Pydantic schema、Alembic migration、服务层和可见测试证据，不接入真实 Agent，不开发前端，不引入 FastAPI。

Tech Stack: Python 3.13, Pydantic 2.13.4, SQLAlchemy 2.0.41, Alembic 1.16.1, PostgreSQL, pytest 8.4.0. LangGraph 1.2.4 在第二阶段接入。

1. 为什么先写这份计划

当前项目已经完成产品需求文档 v2.0、Agent 系统设计 v1.0、技术架构与数据模型设计 v1.0，以及旧后端 P1 工程骨架。问题在于旧后端代码仍围绕 EvidenceCard、EventCluster、ContentArtifact、QualityGateResult、Brief 等早期概念运行，而新版核心链路已经调整为：

SourceSignal
-> EventCandidate
-> EventDossier
-> ReviewResult
-> PublishedEvent

因此下一步不能继续堆旧 HN 闭环，也不应该立刻写真实 Agent。正确顺序是先把新版数据底座立住，否则后面的 LangGraph、多 Agent、后台监控和前端页面都会接在错误对象上。

2. P1 总路线图

阶段名称目标交付物

3. P1-1 范围

必须完成

  • 新增 pydantic 依赖。
  • 建立新版 Pydantic schema。
  • 重构 SQLAlchemy ORM 为新版核心表。
  • 用新的 Alembic migration 替换旧表结构。
  • 建立服务层，保证后续 Agent 只能输出结构化建议，数据库写入由 service 执行。
  • 建立运行记录、Agent 运行记录和发布快照写入能力。
  • 更新测试记录，让人能看到测了什么、用什么测、结果是什么。

不做

  • 不写真实 LLM prompt，不接入 LangGraph，不写三类 Agent 的真实逻辑。
  • 不开发 Next.js 前端或后台 UI。
  • 不引入 FastAPI、Redis、队列、对象存储、向量库。
  • 不做完整事实核验，不继续维护旧 BriefWriterAgent 主链路。

8. 阶段验收标准

  • worker.schemas 可以被导入。
  • worker.models.Base.metadata.tables 只包含新版 P1 核心表，不包含旧主链路表。
  • 新 migration 的 head 是 202606120001。
  • SignalService 可以幂等写入来源和信号。
  • RunLogService 可以记录 pipeline run 和 agent run。
  • EventService 可以完成候选事件、稿件、审稿和发布快照写入。
  • 同一个 candidate 重复发布不会产生重复 published_events。
  • pytest -v 通过，或失败原因被明确记录且不来自本阶段改动。
  • 验收文档写明测了什么、用什么数据测、命令是什么、结果是什么。
