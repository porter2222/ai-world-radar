5. 响应契约

事件列表返回 id、slug、标题、卡片摘要、详情摘要、分类、热度标签、封面图、首页排序位、发布时间，以及首页卡片单行展示用的 source_hint / source_count。前台列表不返回 ranking_score、heat_score、importance_score 等复杂分数；这些分数只用于内部排序和后台审计。

source_count 按平台去重，不按 source_refs 原始条数计数。例如两个 HN 链接加一个 GitHub 链接，列表返回 source_hint="Hacker News 等 2 源"、source_count=2。前台列表不返回完整 source_refs，详情页继续由 GET /events/{slug} 返回完整来源引用。

事件详情额外返回 detail_body、why_it_matters、follow_up_points 和 source_refs，并且必须通过 published_events.dossier_id 联查发布时对应的 event_dossiers 版本。

GET /admin/review-queue 只返回当前仍处于 manual_review 的事件；同一 candidate 存在多版 dossier / review 时，只返回最新 dossier 和最新 review。

Agent run 审计默认不暴露完整 llm_raw_text，只暴露耗时、重试次数、模型信息、token usage、状态和错误摘要。

7. 验收标准

  • 默认不会触发采集、Agent 或发布。
  • 事件列表只返回 published 状态。
  • 事件列表返回首页卡片可用的 source_hint / source_count，但不暴露完整 source_refs。
  • 事件详情按 slug 查询，缺失时行为明确。
  • 后台审计能查看 pipeline run 和 agent run 摘要。
  • manual_review 队列可查询。
  • worker 全量 pytest 通过，PostgreSQL 可读取已发布事件。

8. 执行结论

P1-6 后端产品接口层已完成最小闭环：查询服务、响应 schema、FastAPI 只读 HTTP endpoint、脚本化 PostgreSQL 只读 smoke、真实 HTTP 运行验收、全链路真实验收和验收文档均已落地。

当前可以进入产品页面联调或前端页面开发。FastAPI 在 P1-6 只提供只读产品接口，不触发采集、Agent、发布或重跑。管理端接口当前只服务本地开发和后台页面联调，公开部署前必须补鉴权。
