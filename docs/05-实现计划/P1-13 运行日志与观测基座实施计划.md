# P1-13 运行日志与观测基座实施计划

更新时间：2026-06-26

## 1. 目标

P1-13 的目标是把 `scripts/run_daily_pipeline.py` 从“运行结束后只给一个 JSON summary”升级为“运行过程中实时可观察”。

第一版必须做到：

- 终端实时输出中文阶段日志。
- 同时写入 `runtime/daily-pipeline-latest.log` 和 `runtime/daily-pipeline-latest.jsonl`。
- 每次运行生成带时间戳的 `.log` / `.jsonl` 文件。
- 日志 timestamp 统一转换为本地时间 `Asia/Shanghai`，JSONL 中必须带 `+08:00`。
- 覆盖 daily pipeline、source 采集、selector、LangGraph workflow、Agent、LLM 请求、结构化校验、工具调用、AgentRun 入库、失败、重试、fallback 和 30 秒心跳。
- 日志脱敏，不能把 API key、数据库密码、Cookie、Authorization 等敏感信息写入终端、文件或文档。
- 日志 Sink 失败不能拖垮主流程。

本阶段不做：

- 后台管理页面。
- Redis / 消息队列。
- OpenTelemetry 全量接入。
- 日志检索服务。
- 完整 prompt / response 默认落盘。
- source 网络失败治理。
- DailyEdition、历史归档页或旧 `EvidenceCard / EventCluster / Brief` 主链路。

## 3. 核心设计

### 3.1 RunLogger

`RunLogger` 是唯一业务日志入口，统一生成结构化 `LogEvent`，并分发到多个 Sink。

第一版字段包括：

- `timestamp`
- `level`
- `run_id`
- `trace_id`
- `span_id`
- `parent_span_id`
- `component`
- `stage`
- `event`
- `status`
- `message_zh`
- `duration_ms`
- `source_key`
- `candidate_group_id`
- `pipeline_run_id`
- `agent_run_id`
- `agent_name`
- `tool_name`
- `provider`
- `model`
- `retry_count`
- `token_usage`
- `counts`
- `error_type`
- `error_message`
- `metadata`

### 3.2 Sink

第一版实现三个 Sink：

- `ConsoleSink`：输出中文终端日志。
- `TextFileSink`：输出人类可读 `.log`。
- `JsonlFileSink`：输出结构化 `.jsonl`。

`create_daily_pipeline_logger()` 会同时创建：

```text
runtime/daily-pipeline-latest.log
runtime/daily-pipeline-latest.jsonl
runtime/daily-pipeline-YYYYMMDD-HHMMSS.log
runtime/daily-pipeline-YYYYMMDD-HHMMSS.jsonl
```

### 3.3 本地时间

日志内部明确使用：

```python
LOCAL_TZ = ZoneInfo("Asia/Shanghai")
```

`RunLogger._timestamp()` 会把 aware datetime 转换到 `Asia/Shanghai`；naive datetime 按本地时间处理。验收要求是 JSONL 中全部 `timestamp` 以 `+08:00` 结尾。

### 3.4 心跳

`RunLogger.stage(..., heartbeat_interval_seconds=30)` 会在长阶段持续输出：

```text
[仍在运行：collect_sources]
```

本轮真实运行发现一个重要问题：当 stage 本身传入 `metadata` 时，heartbeat 也传入 `metadata={"elapsed_seconds": ...}`，会导致 `metadata` 重复传参。已新增回归测试并修复为合并 metadata。

## 7. 当前结论

P1-13 运行日志与观测基座已完成第一版：

- 手动运行时能实时看到中文阶段日志。
- 定时任务未来可复用同一套 `runtime/*.log` 和 `runtime/*.jsonl`。
- Agent / LLM / tool / retry / fallback / AgentRun 入库均有可追踪事件。
- 真实运行已验证日志可定位 source 失败、selector 运行、LLM 结构校验失败与 repair retry。
- 日志时间已按用户要求统一转换为本地时间 `Asia/Shanghai`。
