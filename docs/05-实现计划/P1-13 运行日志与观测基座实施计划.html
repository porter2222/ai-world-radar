P1-13 运行日志与观测基座实施计划

更新时间：2026-06-26

P1-13 已把 run_daily_pipeline.py 升级为实时可观察的运行链路：终端中文日志、runtime 文本日志、JSONL 结构化事件、Agent/LLM/tool 日志、30 秒心跳和本地时间 +08:00 均已完成并通过真实运行验收。

2. 实现范围

新增 worker/observability/run_logger.py，并接入 CLI、DailyPipelineService、selector、LangGraph workflow、EventPipelineTools、LLMJsonAgent、三类 LLM Agent 和相关测试。

第一版不做后台管理页、Redis、消息队列、OpenTelemetry、日志检索服务，也不默认保存完整 prompt / response。

3. 核心设计

RunLogger + Sink

业务代码
  -> RunLogger.emit(event)
      -> ConsoleSink
      -> TextFileSink
      -> JsonlFileSink

每条事件包含 run_id、trace_id、span_id、component、stage、event、message_zh、duration_ms、agent_name、tool_name、retry_count、token_usage、counts、error_type 和 metadata 等字段。

本地时间

日志统一使用 ZoneInfo("Asia/Shanghai")。JSONL 验收结果为 bad_timezone=0，说明全部 timestamp 均以 +08:00 结尾。
